Findings are automatically grouped by semantic similarity using content-weighted TF-IDF (bigrams) + Bisecting K-Means. The optimal cluster count is determined automatically via Gap Statistic + Silhouette — no fixed k assumed. Gap statistic compares actual inertia against reference distributions; silhouette measures geometric separation. Combined score: 0.55×gap + 0.45×silhouette − 0.002k. You can also override k manually below.

SUGGESTED QUERIES

Add New Finding

Findings are clustered automatically based on all fields you provide. The Model Theme field, while optional, helps the clustering algorithm group findings more accurately.

ℹ Clustering is performed automatically

After submission, the system will re-run semantic clustering across all findings and assign this finding to the most appropriate cluster (C1, C2, …) based on content similarity.

Paste CSV Data

Paste rows with columns: Finding ID, Model Theme, Title, Description, Business Justification and optionally Suggested Remediation. One row per line. No header row needed. Model Theme and Suggested Remediation are optional.

Paste JSON Array

Array of objects with keys: finding_id, model_theme (optional), title, description, business_justification, suggested_remediation (optional).

Upload Excel or CSV File

Upload an .xlsx or .csv file. Column order does not matter. Required and optional columns listed below:

Column Header Required Description
Finding ID Required Unique identifier e.g. F013
Model Theme Optional Risk category e.g. Model Governance
Title Required Short descriptive title
Description Required Full detail of the issue observed
Business Justification Required Business & regulatory rationale
Suggested Remediation Optional Action steps to resolve the finding

Rows with duplicate Finding IDs or missing required fields will be skipped automatically.
